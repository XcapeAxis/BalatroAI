# Self-Supervised Learning Overview

BalatroAI uses self-supervised learning to reuse large trace corpora when high-quality expert labels are sparse.
Instead of directly learning action logits from teacher labels, the P36 pipeline first learns a reusable representation from replay steps, then feeds that representation into downstream BC/DAgger/RL paths.

## Why This Exists

- trace volume is large (sim/oracle and real-aligned artifacts), while curated expert actions are comparatively expensive
- representation pretraining reduces cold-start instability for later policy optimization
- the same replay contract can be reused across research tasks without touching core engine code

## Replay Contract (P36 v1)

Unified replay rows are generated by `trainer/replay/*` with a stable step schema:

- identity: `replay_id`, `episode_id`, `step_id`, `run_id`, `source`
- action: `phase`, `action_type`, `action_payload`
- observability: `state_hash_*` scopes and `score_delta`
- dynamics: `resources_delta` (`chips/mult/money/hands/discards` deltas)
- training gate: `valid_for_training`, `invalid_reason`

`valid_for_training=true` is required by default in replay self-supervised training.

## Current Objectives

P36 v1 includes lightweight objectives in `trainer/experiments/selfsup_objectives.py`:

1. `mask`: masked action-token reconstruction
2. `next_delta`: next-step `score_delta` regression
3. `hybrid`: weighted combination of the two objectives

These are intentionally small and stable for plumbing validation, not peak performance.

## How To Run

Build replay dataset (real + sim roots):

```powershell
python -B -m trainer.replay.storage --real-roots docs/artifacts/p32 docs/artifacts/p13 --sim-roots sim/tests/fixtures_runtime docs/artifacts/p32/smoke_position_fixture --out-dir docs/artifacts/p36/replay/smoke_latest --max-episodes-per-source 6
```

Run replay self-supervised smoke:

```powershell
python -B -m trainer.experiments.selfsup_train --config configs/experiments/p22_selfsup_smoke.yaml
```

Run P22 quick matrix (includes existing self-supervised rows):

```powershell
powershell -ExecutionPolicy Bypass -File scripts/run_p22.ps1 -Quick
```

## Integration With BC / DAgger

- BC/DAgger training remains unchanged as the production baseline path
- replay self-supervised checkpoints are currently optional pretrain artifacts
- future work can initialize policy backbones from replay-pretrained encoders

## Limits

- v1 focuses on structure and reproducibility; it is not a full RL/self-play stack
- policy gains are not guaranteed without downstream finetuning/evaluation
- trace quality still governs representation quality; invalid steps are filtered by contract checks
