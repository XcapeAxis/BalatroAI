# MVP Demo 演示脚本

## 开场 15 秒

可以直接这样讲：

“这是一个本地 AI 决策沙盘，不需要原版游戏窗口。左边选场景，中间看局面，右边看模型和基线怎么给建议，下面直接看执行这一步以后会发生什么。页面里的模型建议来自真实训练出的 checkpoint，训练过程本身也能在页面里看到。”

## 建议演示顺序

1. 高收益起手
2. 高压弃牌转折
3. Joker 协同爆发

这个顺序的原因：

- 先用最直观的场景建立信任。
- 再展示高风险局面下，AI 为什么不能只看眼前筹码。
- 最后用 Joker 协同把“可解释推荐”讲完整。

## 场景 1：高收益起手

要讲什么：

- 这是最适合开场的场景。
- 局面容易看懂，推荐动作和收益变化都很明显。
- 适合证明 Demo 不是静态卡片，而是真的在驱动本地模拟器。

推荐演示动作：

1. 载入 `高收益起手`。
2. 先指中间局面，再指右侧 `模型 vs 基线首选`。
3. 说明两边首选一致，重点就不是“谁赢”，而是“执行后会发生什么”。
4. 点击“执行当前动作”。
5. 展示局面高亮、结果预览、时间线一起变化。

建议话术：

“这个场景最适合开场，因为所有人一眼都能看懂。页面不是只给一张推荐卡，而是把局面、推荐和执行后的变化直接连起来了。”

## 场景 2：高压弃牌转折

要讲什么：

- 这个场景最适合展示“AI 不是只看眼前筹码”。
- 当前资源已经很紧，错误动作代价高。
- 重点是弃牌与后续机会的权衡。

推荐演示动作：

1. 切到 `高压弃牌转折`。
2. 先指出 `手数 1 / 弃牌 1` 和 `高风险`。
3. 再看右侧首选卡，强调“这一步不收分，但在换后手”。
4. 对比模型和基线的备选差异，说明为什么不能只看眼前筹码。
5. 可以点一次“自动演示 3 步”或者手动执行。

建议话术：

“这个场景能说明助手不是机械追求即时收益，而是在做资源调度。它会考虑现在打出去值不值，以及如果先弃牌，能不能换来更高质量的下一手。”

## 场景 3：Joker 协同爆发

要讲什么：

- 这是“可解释 AI”最强的场景。
- 推荐不仅告诉你打什么，还告诉你为什么收益会被放大。
- 适合强调 Joker 对预期得分的影响。

推荐演示动作：

1. 切到 `Joker 协同爆发`。
2. 先看中间 `Joker / 机制` 面板。
3. 再看右侧首选卡和下面结果区。
4. 说明这一步为什么会被 Joker 放大。
5. 执行一次动作，让时间线补上解释链路。

建议话术：

“这里不只是排卡强弱，而是在解释局面里的特殊机制如何改变动作价值。这也是我把它做成产品页面，而不是调试页的原因。”

## 如何展示训练面板

训练面板是这页可信度最高的一块，建议一定展示。

可以按这个顺序讲：

1. 先看“成果总结”，说明这版模型已经接管页面推荐。
2. 指出当前模型的 `run_id` 和样本规模。
3. 指出 `Val Loss / Top-1 / Top-3`。
4. 展示 loss 曲线和右侧的场景对齐结果。
5. 说明 UI 内可以直接发起新训练。
6. 如果现场允许，点一次“快速烟雾训练”展示状态流转。

建议话术：

“我没有把训练放在命令行里藏起来，而是把它产品化了。面试官可以直接看到当前训练到了哪一轮、loss 怎么变化、当前最佳 checkpoint 是哪一个。”

## 如何正确描述模型

建议明确这样说：

- 这是一个真实训练出的最小可用监督模型。
- 它不是最终最强代理。
- 当前重点是把训练、推理、解释、可视化打通成一个可演示产品。
- unsupported phase 仍会回退到启发式，这是有意为之的 MVP 收敛。

不要这样说：

- “已经接近完整 Balatro 通用最优代理”
- “可以完全替代原版客户端”
- “已经做完长期主线”

## 现场如果要触发训练

推荐顺序：

1. 先点“快速烟雾训练”。
2. 给面试官看状态从 `排队中 -> 构建数据集 -> 正式训练 / 评估`。
3. 指一下训练面板里的进度条、曲线和最新状态。
4. 如果时间充裕，再说明也支持 2 小时正式训练。

如果要讲 2 小时版本：

- 可以说“这个按钮会在后台启动一轮更像正式版本的训练预算，用于生成新的展示模型”。
- 不建议在时间非常紧的现场从零开始完整跑完 2 小时。

## fallback 方案

优先级建议：

1. 重新运行 Demo

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_mvp_demo.ps1 -OpenBrowser
```

2. 如果需要先补最小模型，再启动

```powershell
powershell -ExecutionPolicy Bypass -File scripts\bootstrap_mvp_demo.ps1 -OpenBrowser
```

3. 如果浏览器状态不稳，直接展示 fallback 截图

- `docs/artifacts/mvp/fallback/basic_play_demo.png`
- `docs/artifacts/mvp/fallback/high_risk_discard_demo.png`
- `docs/artifacts/mvp/fallback/joker_synergy_demo.png`

4. 如果需要非交互证据，展示这些产物

- `docs/artifacts/mvp/training_status/latest.json`
- `docs/artifacts/mvp/model_train/latest_run.txt`
- `docs/artifacts/mvp/api_smoke_20260309_154858.json`
- `docs/artifacts/mvp/fallback_assets_20260309_144449.md`

## 面试前 30 秒检查清单

- Demo 页面已打开。
- 主题已切到你更习惯展示的模式。
- 当前场景先停在 `高收益起手`。
- 训练面板可见。
- fallback 截图窗口已打开。
- `latest_run.txt` 和 `training_status/latest.json` 可随时切出来证明模型和训练都是真实产物。
