# MVP Demo 演示脚本

> English version: [MVP_DEMO_SCRIPT.en.md](MVP_DEMO_SCRIPT.en.md)

## 开场 15 秒

可以直接这样讲：

“这是一个本地 AI 决策沙盘，不需要原版游戏窗口。默认打开是薄顶栏加左中右三栏：左边选场景，中间看局面，右边看模型和基线怎么建议，下方用摘要带看结果、时间线和训练成果。宽屏优先一屏看全，窗口变窄时也会自动收成更稳的布局。”

## 第一屏先怎么讲

建议固定这条顺序：

1. 先停在默认首页
2. 用一句话点出页面动线
3. 先讲当前场景和局面
4. 再讲 `模型 vs 启发式`
5. 最后点执行或切底部摘要标签页

推荐话术：

“这页不是把很多面板堆在一起，而是先让人一眼扫完整个决策链。先看场景，再看局面，再看建议，最后看执行后会发生什么。”

## 建议演示顺序

1. 高收益起手
2. 高压弃牌转折
3. Joker 协同爆发

原因很简单：

- 先用最直观的场景建立信任
- 再用高风险场景解释为什么 AI 不能只看眼前收益
- 最后用 Joker 协同把“可解释推荐”讲完整

## 场景 1：高收益起手

重点讲：

- 这是最适合开场的场景
- 一步兑现收益，非常直观
- 适合先证明页面不是静态卡片，而是真的会驱动本地模拟器

推荐动作：

1. 载入 `高收益起手`
2. 先指中间局面，再指右侧 `模型 vs 启发式`
3. 强调两边首选一致，说明这个场景重点不是“谁赢”，而是“执行后如何联动变化”
4. 点击 `执行当前动作`
5. 展示局面、结果预览、时间线同步变化

建议话术：

“这个场景最适合开场，因为所有人都能一眼看懂。点击一次之后，中间牌面、下方结果和时间线会一起变，这就是这页最重要的记忆点。”

## 场景 2：高压弃牌转折

重点讲：

- 这个场景最适合展示“AI 不是只看眼前筹码”
- 当前资源已经很紧，错误动作代价高
- 模型和基线首选一致，但备选已经开始分开

推荐动作：

1. 切到 `高压弃牌转折`
2. 先指出 `手数 1 / 弃牌 1` 和 `高风险`
3. 再看右侧首选卡，强调“这一步不收分，但在换后手”
4. 指出模型和启发式的备选差异
5. 再执行一次，展示因果预览

建议话术：

“这里最能说明模型不是摆设。虽然首选一致，但备选已经开始分开，说明模型和基线对‘如果第一步走不通，第二步怎么补’的理解不一样。”

## 场景 3：Joker 协同爆发

重点讲：

- 这是“可解释 AI”最强的场景
- 推荐不仅告诉你打什么，还告诉你为什么收益会被放大
- 适合强调 Joker 如何改变动作价值

推荐动作：

1. 切到 `Joker 协同爆发`
2. 先看中间 `Joker / 机制`
3. 再看右侧首选卡和下方结果区
4. 说明这一步为什么会被 Joker 放大
5. 执行一次动作，让时间线补上解释链

## 什么时候切底部摘要标签页

摘要带不要一上来就频繁切，适合在这三种时刻再切：

- 要展开讲 `一步结果预览`
- 要展开讲 `训练成果`
- 要让面试官盯住某一块，而不是继续扫全景

推荐话术：

“首页负责一眼看懂，底部摘要带负责把结果、时间线和训练成果分别讲透。”

## 如何讲训练成果

训练区建议按这个顺序讲：

1. 先看 `成果总结`
2. 指出当前 `run_id / checkpoint`
3. 再看样本规模、最佳验证损失、Top-1、Top-3
4. 最后看训练曲线和场景对齐结果

建议话术：

“我没有把训练藏在命令行里，而是直接把它做成可展示的成果面板。这里能看到当前模型来自哪个 checkpoint、用了多少样本、指标是多少，也能直接看到曲线。”

## 如何正确描述模型

建议明确这样说：

- 这是一个真实训练出的最小可用监督模型
- 它已经接管了 Demo 的推荐
- 它不是最终最强代理
- 当前重点是把训练、推理、解释和可视化打通成一个可演示产品

不要这样说：

- “已经接近完整 Balatro 通用最优代理”
- “可以完全替代原版客户端”
- “长期主线已经做完”
